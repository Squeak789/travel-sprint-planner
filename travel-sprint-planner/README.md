# 疾行旅策 SprintTrip

一个面向“特种兵式旅游”需求的路线规划网站原型。

## 当前能力

- 用户可直接输入城市名称，不再依赖固定城市下拉框。
- 后端会先让 AI 选出该城市的热门景点，并自动把其中的必打卡点填入。
- 根据城市、天数、出发收工时间、强度模式生成高密度路线。
- 支持勾选必须打卡景点，算法优先安排。
- 支持接入高德 API 或百度 API，自动获取地理坐标和真实路径时长。
- 当地图接口失败或无 Key 时，自动回退为估算时长，保证可用性。
- 可选接入兼容 OpenAI Chat Completions 的 AI 接口，用于热门攻略增强与路线优化建议。

## 快速使用

1. 用浏览器打开 index.html。
2. 输入城市、旅行天数、时间窗口、强度模式。
3. 启动后端优化服务：
	- 进入 backend 目录
	- 运行 npm install
	- 运行 npm run dev
4. 回到页面，填入起点位置、地图服务商和地图 Key（建议至少配置高德 Key）。
5. 点击“AI 获取热门景点”，让后端基于城市生成热门景点包并自动选出必打卡点。
6. 勾选或调整必须打卡景点，点击“生成特种兵路线”。
7. 如需攻略增强，勾选“启用 AI 热门攻略增强”，并填写后端优化服务地址、AI 接口地址、模型和 Key。

## 地图 API 说明

- 高德：使用 Web 服务接口（地点搜索 + 路径规划）。
- 百度：使用 Place Search + DirectionLite 接口。
- 由于本项目是纯前端，Key 会暴露在浏览器网络请求里。
- 生产环境建议使用后端代理 API，请勿在前端直接暴露高权限密钥。

## AI 说明

- AI 调用已迁移到后端，由后端统一采集热门信号、筛选热门景点并参与量化优化。
- 前端只负责提交参数，不再直接请求 AI。
- 默认采用 OpenAI 兼容格式，示例地址为 OpenRouter。
